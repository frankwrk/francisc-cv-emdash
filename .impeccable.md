## Design Context

### Users
Fellow developers, future collaborators and employers, and anyone who finds systems thinking and craft-focused writing engaging or educational. They are technically literate, aesthetically aware, and will notice the details. They visit both to read and to evaluate — the site must earn trust on both dimensions simultaneously.

### Brand Personality
**Precise. Bold. Opinionated.**

The site should feel like it was built by someone who cares deeply about craft — not "look how hard I worked" but "look how clearly I think." It should inspire systems thinking and design. Every element earns its place. Nothing is decorative by accident.

### Aesthetic Direction
**Dark by default. Light mode available.**

Readers encounter this site in a developer's context — late evenings, focused sessions, the same environment they read documentation and write code. Dark mode is the native state.

**Accent color: `#ECC30B` (warm amber/gold)**
Chosen from the preferred palette {#ECC30B, #E94F37, #C200FB, #002E2C, #06A77D}. Gold/amber signals craft and intentionality without the aggression of red or the tech-cliché of teal. In OKLCH: approximately `oklch(82% 0.17 88)`. Used in one place at a time — never decoratively, always meaningfully (links on hover, active states, a single highlight, code callouts).

**References:**
- [micahcarroll.com](https://micahcarroll.com/) — bold portfolio work grids, motion thumbnails, strong visual hierarchy
- [jakub.kr](https://jakub.kr/) — restrained layouts, detailed writing, interfaces that reward close reading

**Anti-references:** Generic AI-generated portfolios — rounded cards, glassmorphism, gradient text, hero metrics. Must not look like it was produced by a prompt.

### Font System (Geist)

Three roles within the Geist family:

| Role | Face | Weight | Usage |
|------|------|--------|-------|
| Display | Geist Sans | 700–900, tight tracking (−0.04em) | H1 headings, hero text, project titles |
| Body | Geist Sans | 400–500, normal tracking | All prose, navigation, UI labels |
| Mono | Geist Mono | 400–500 | Code blocks, figure captions, technical metadata, timestamps, tags |

**Scale (fixed rem, dark UI context):**
- `--text-xs`: 0.75rem / Mono — timestamps, tags, labels
- `--text-sm`: 0.875rem — secondary UI, captions, metadata
- `--text-base`: 1rem — body prose
- `--text-lg`: 1.125rem — lead paragraphs, article intros
- `--text-xl`: 1.375rem — section headings (H3)
- `--text-2xl`: 1.75rem — article titles in lists (H2)
- `--text-3xl`: 2.25rem — page-level H2
- `--text-4xl`: 3rem — H1 on content pages
- `--text-5xl`: 4rem / tight tracking — hero display text
- `--text-6xl`: 5.5rem / ultra-tight tracking (−0.05em) — homepage statement, large project headings

Line heights: `1.1` for display, `1.3` for headings, `1.65` for body prose. Max line length: `68ch`.

### Design Principles

1. **One accent, one moment.** `#ECC30B` appears in exactly one place per view. Overuse kills it.
2. **Mono signals precision.** Geist Mono is not just for code — it marks anything technical, timestamped, or labeled. It creates a quiet information layer beneath the display text.
3. **Hierarchy through contrast, not decoration.** Size, weight, and space do the work. No borders, shadows, or icons as substitutes for clear structure.
4. **Accessibility and SEO as architecture.** Semantic HTML, meaningful alt text, structured data (JSON-LD), and logical heading order are not afterthoughts — they define the content structure from the start.
5. **Motion reveals, never decorates.** Entrances, state changes, and loading indicators only. No ambient animation.
6. **Dark first, light faithful.** The dark theme is the primary experience. Light mode must be equally considered — not an inversion, but a reinterpretation using the same palette logic.
